## 🟠 Bugs sérios

### S-1 · 10 views CRUD sem a barra de botões padronizada
**Severidade:** 🟠 Sério — impacta UX e percepção de produto pronto.
**Fase:** 2 / 7.
**Reprodução:** Logado em Org A, executar via console:
```js
const viewMap = {'sold':'vsold','cal':'vcal','pend':'vpend','com':'vcom','maint':'vmaint','rdo':'vrdo','quality_reports':'vquality_reports','scaf':'vscaf','civil_sinapi':'vcivil_sinapi','elec_specs':'velec_specs','equip':'vequip'};
// Para cada uma, verificar bc-row → resultado: ausente ou incompleta
```
**Resultado:** As views abaixo NÃO mostram o trio `[Excel ▾] [Cadastrar via IA] [+ Novo]`:

| View | Falta |
|---|---|
| `sold` (Soldadores) | tudo |

### S-3 · `VIEW_TABLE_MAP` em escopo IIFE não exposto + `openImportExcel(viewkey)` falha silenciosamente
**Severidade:** 🟠 Sério — usuário clica sem feedback.
**Fase:** 3.
**Evidência:**
```js
typeof window.VIEW_TABLE_MAP // "undefined"

window.openImportExcel('mat') // não throwsmas modal não abre
```
**Impacto:** Em algumas viewkeys (não mapeadas no map interno), `openImportExcel` retorna em silêncio. Sem `console.warn`, sem toast.

### S-4 · 162 tabelas com exposição `pg_graphql_anon_table_exposed`
**Severidade:** 🟠 Sério (depende se GraphQL endpoint está exposto).
**Fase:** 9.
**Evidência:** Supabase advisor security `pg_graphql_anon_table_exposed` × 162 tabelas (ex.: `ai_extractions`, `anonymous_ia_usage`, `materials_catalog`, …). Role `anon` tem `SELECT` GRANT pelo wrapper GraphQL.

**Nota:** A RLS bloqueia leituras anônimas, então não é vazamento direto, MAS amplia a superfície de ataque se uma policy for mal configurada no futuro.

**Recomendação:** Desabilitar GraphQL anon access ou, em cada tabela sensível, executar `REVOKE SELECT ON <tabela> FROM anon;`.

## 🟡 Bugs menores / UX

### M-1 · `meta-robots: noindex,nofollow` em produção comercial
Bloqueia indexação Google. Remover do `index.html` e `hydrostec_v9.html` quando for go-live público.

### M-2 · Mensagem "Você está abrindo via arquivo direto" aparece em hospedagem Web
String "Pra login automático entre o sistema principal e o HUB Planejador, feche esta janela e dê duplo-clique em `ABRIR_PROJECT_IA.bat`…" aparece na hospedagem Vercel. Mensagem é válida só para `file://` local.

### M-3 · `vercel.json` sem `Content-Security-Policy` nem `Strict-Transport-Security` explícita
Vercel fornece HSTS por default mas é boa prática declarar. Para SaaS comercial, CSP é recomendada (script-src próprio + CDNs autorizados).

### M-4 · 5 tabelas com `roles=public` em policies que deveriam ser `authenticated`
`civil_concrete_pours`, `equipments`, `maint_work_orders`, `painting_inspections`, `scaffolds`. Funcionalmente equivalente (USING usa `auth.uid()`) mas inconsistente com o padrão das demais tabelas. Trocar para `TO authenticated`.

### M-5 · 304 unused_index + 190 multiple_permissive_policies + 3 unindexed_foreign_keys
Advisor performance do Supabase. Sem impacto imediato com volume baixo, mas ao escalar vira custo de write/cpu. Limpar antes de massificar uso.

**Validação:** Logado como Org B no browser, queries via `window.sb` (que respeita RLS):
```js
await sb.from('projects').select().eq('org_id', ORG_A_ID)         // .length = 0
await sb.from('materials_catalog').select().eq('org_id', ORG_A_ID) // .length = 0
await sb.from('welders').select().eq('org_id', ORG_A_ID)           // .length = 0
await sb.from('pendings').select().eq('org_id', ORG_A_ID)          // .length = 0
await sb.from('organizations').select()                            // só "Auditoria B"
await sb.from('materials_catalog').insert({org_id: ORG_A_ID, …})   // BLOCKED com erro RLS
```

**Resultado:** Isolamento perfeito nas 10 tabelas testadas.
